1. Project Overview

The Store Project is a comprehensive web application for managing an online store. It provides separate interfaces for administrators (store owners) and customers, with role-based access control to ensure appropriate permissions for each user type.

Key Features

  • User Authentication - Secure login and registration system with JWT token-based authentication
  • Role-Based Access - Different interfaces and permissions for administrators and customers
  • Admin Dashboard - Sales analytics, inventory management, and user administration
  • Product Management - Add, edit, and delete products with image uploads
  • Order Processing - Complete order lifecycle management from creation to fulfillment
  • Sales Reporting - Detailed sales reports with filtering options
  • Customer Shopping - Product browsing, cart management, and order history

2. Architecture

The Store Project follows a client-server architecture with a clear separation between frontend and backend components.

Technology Stack

  • Frontend: HTML, CSS, JavaScript, Bootstrap 5
  • Backend: Django (Python web framework)

3. Authentication System

The Store Project implements a secure JWT (JSON Web Token) based authentication system that manages user sessions and API access.

Authentication Flow

  1. User submits credentials (username/password) via login form
  2. Backend validates credentials and generates JWT tokens (access and refresh)
  3. Tokens are stored in browser's localStorage with expiration time
  4. Access token is included in Authorization header for API requests
  5. When access token expires, system automatically attempts to refresh it
  6. On logout, tokens are cleared from localStorage

Key Authentication Functions

The authentication system is implemented in auth.js with these key functions:

  • login(username, password) - Authenticates user and stores tokens
  • logout() - Clears authentication tokens
  • isAuthenticated() - Checks if user is currently authenticated
  • apiRequest(endpoint, options) - Makes authenticated API requests
  • refreshToken() - Refreshes expired access token

Token Storage

Tokens are stored in localStorage with the following structure:

{
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",  // Access token
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", // Refresh token
  "expiresAt": "2023-06-01T12:00:00.000Z"               // Expiration timestamp
}

4. Role-Based Access Control

The application implements role-based access control to direct users to appropriate interfaces and restrict access to authorized features.

User Roles

  • Owner (Admin) - Full access to admin dashboard, product management, sales data, and user management
  • Customer - Access to product browsing, shopping cart, and order management

Role-Based Routing

The role-router.js module handles redirecting users based on their role:

  • Extracts user role from JWT token payload
  • Redirects to appropriate interface based on role
  • Prevents unauthorized access to restricted pages

Key functions:

  • getUserRole() - Extracts role from JWT token
  • redirectBasedOnRole() - Redirects user to appropriate interface
  • checkAccess(allowedRoles) - Verifies if current user has required role

7. API Integration

The frontend communicates with the backend through a RESTful API. All API requests are authenticated using JWT tokens.

API Endpoints

Endpoint Method Description Access
/api/auth/login/ POST Authenticate user and get tokens Public
/api/auth/register/ POST Register new user Public
/api/auth/refresh/ POST Refresh access token Authenticated
/api/products/ GET Get all products Public
/api/products/{id}/ GET Get product details Public
/api/products/ POST Create new product Admin
/api/products/{id}/ PUT Update product Admin
/api/products/{id}/ DELETE Delete product Admin
/api/orders/ GET Get user orders Authenticated
/api/orders/ POST Create new order Authenticated
/api/orders/{id}/ GET Get order details Order Owner/Admin
/api/sales/ GET Get sales data Admin
/api/users/ GET Get all users Admin

Making API Requests

The apiRequest function in auth.js handles all API communication: 

// Example of making an authenticated API request
async function fetchProducts() {
  try {
    const response = await window.Auth.apiRequest('/products/');

    if (!response.ok) {
      throw new Error('Failed to fetch products');
    }

    const products = await response.json();
    displayProducts(products);
  } catch (error) {
    console.error('Error fetching products:', error);
  }
}

9. Navigation & URLs

The Store Project is accessed through Django's static file serving. All frontend pages are accessible via specific URL patterns.

URL Structure

When the Django development server is running, the application is accessible at localhost:8000. The frontend pages are served from the /static/ path.

Page URL Description
Landing Page http://localhost:8000/static/index.html Main entry point to the application
Login http://localhost:8000/static/login.html User authentication page
Registration http://localhost:8000/static/register.html New user registration
Documentation http://localhost:8000/static/documentation.html This documentation page

Admin Interface URLs

The admin interface is accessible through the following URLs:

Page URL Description
Dashboard http://localhost:8000/static/admin/dashboard.html Admin dashboard with analytics
Products http://localhost:8000/static/admin/products.html Product management
Users http://localhost:8000/static/admin/users.html User management
Sales http://localhost:8000/static/admin/sales.html Sales management and reports
Purchases http://localhost:8000/static/admin/purchases.html Purchases management

Customer Interface URLs

The customer interface is accessible through the following URLs:

Page URL Description
Products http://localhost:8000/static/customer/products.html Product browsing and shopping
Orders http://localhost:8000/static/customer/orders.html Order history and tracking

Navigation Flow

The application implements role-based navigation:

  1. Users start at the landing page (/static/index.html)
  2. Unauthenticated users are directed to login
  3. After authentication, users are redirected to their role-specific interface:
    • Administrators → Admin Dashboard
    • Customers → Product Browsing

10. Setup & Deployment

Instructions for setting up and deploying the Store Project.

Prerequisites

  • Python 3.8+ with pip
  • Database (sqllite3)

Backend Setup

  1. Clone the repository
  2. Create and activate a virtual environment
  3. Install dependencies: pip install -r requirements.txt
  4. Configure database settings in settings.py
  5. Run migrations: python manage.py migrate
  6. Create superuser: python manage.py createsuperuser
  7. Start development server: python manage.py runserver

Frontend Access

The frontend is served directly through Django's static file serving:

  1. Start the Django development server: python manage.py runserver
  2. Access the application at http://localhost:8000/static/index.html
  3. No separate frontend server is needed as all static files are served by Django

Note: For development purposes, you can modify the HTML, CSS, and JavaScript files directly in the static/ directory. Django will serve the updated files without requiring a server restart.

Deployment

For production deployment:

  1. Set DEBUG = False in settings.py
  2. Configure ALLOWED_HOSTS in settings.py
  3. Set up a production database
  4. Configure static files serving
  5. Set up a web server (Nginx, Apache) with WSGI/ASGI
  6. Configure SSL for secure connections
